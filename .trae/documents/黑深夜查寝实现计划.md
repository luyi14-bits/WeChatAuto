# 黑深夜查寝 - 定时发送照片功能实现计划

## 一、需求总结

用户在晚上11点左右拍好查寝照片 → 设置定时时间（如凌晨1-2点）→ 程序在设定时间自动将照片发送到微信群，可附带自定义文字说明。

- **照片来源**：选择已有照片文件（手机拍好传到电脑）
- **任务模式**：支持多任务队列，每张照片可设定不同时间、不同群
- **附加文字**：支持自定义文字说明
- **品牌标识**：改为 `luyi14og`

## 二、当前状态分析

项目是一个 391 行的 Python 单文件 `WeChatAuto.py`，基于 PyQt5 + uiautomation。

**已有能力：**
- ✅ Windows 注册表自动寻址微信安装路径
- ✅ 微信进程检测与自动启动
- ✅ UIA 微信窗口操控（搜索联系人、发送文本、粘贴文件）
- ✅ PyQt5 GUI（双选项卡：发送 + 设置/日志）
- ✅ 适老化大字体样式
- ✅ QTimer 每秒轮询框架（`check_scheduled_tasks` 目前为空）
- ✅ 多线程 WorkerThread（防界面冻结）

**缺失能力：**
- ❌ 定时任务队列数据结构
- ❌ 定时任务管理 UI（添加/查看/删除/列表展示）
- ❌ `check_scheduled_tasks()` 实际实现
- ❌ 任务持久化（关程序后任务丢失）
- ❌ 拍照功能（不需要，用户用手机拍好传到电脑）

#### 修改5：任务数据结构

```python
@dataclass
class ScheduledTask:
    id: int          # 唯一标识
    recipient: str   # 接收人
    message: str     # 文字内容
    file_path: str   # 照片路径
    send_time: datetime  # 预定发送时间
    status: str      # "等待中" / "已发送" / "发送失败"
    error_msg: str   # 错误信息（失败时）
```

#### 修改6：`check_scheduled_tasks()` 实现

每秒轮询逻辑：
1. 遍历 `self.scheduled_tasks`
2. 找到 `status == "等待中"` 且 `send_time <= datetime.now()` 的任务
3. 标记为 `"发送中"`，创建 `WorkerThread` 执行发送
4. 发送成功 → 标记 `"已发送"`，更新表格
5. 发送失败 → 标记 `"发送失败"`，记录错误信息
6. 每个周期最多发送1个任务，防止同时多个任务冲突

#### 修改7：任务队列操作方法

- `add_scheduled_task(recipient, message, file_path, send_time)` — 添加任务
- `remove_scheduled_task(task_id)` — 删除任务
- `refresh_task_table()` — 刷新表格显示
- `log(msg)` — 现有方法，增加日志记录

#### 修改8：任务持久化（可选但推荐）

使用 Python 内置 `json` 模块，将任务队列保存到 `tasks.json`：
- 窗口关闭时自动保存
- 窗口打开时自动加载
- 只保存 `"等待中"` 状态的任务，已完成的丢弃

## 七、边界情况与容错

1. **定时时间已过期**：添加任务时校验，若时间已过弹出警告但允许添加（用户可能调到第二天）
2. **文件已被删除**：发送前再次检查文件是否存在，不存在则标记失败
3. **微信未登录**：到点执行时若微信未运行，自动启动微信并等待
4. **程序关闭时任务还在**：JSON 持久化保存，下次启动恢复
5. **同一时间多个任务**：逐个串行执行，每个发送完再发下一个
6. **重复添加**：不阻止重复，由用户自行管理
