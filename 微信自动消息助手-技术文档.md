# 微信自动消息助手 — 技术文档

## 1. 项目概述

微信自动消息助手是一款基于 PyQt5 + wxauto + QProcess 的 Windows 桌面应用程序，支持即时发送与定时循环发送微信消息、图片和文件。核心设计采用**进程隔离架构**，通过独立子进程执行 wxauto 自动化操作，彻底消除 COM 线程冲突导致的闪退问题。

### 核心技术栈

| 层级 | 技术 | 用途 |
|------|------|------|
| 自动化引擎 | [wxauto](https://github.com/cluic/wxauto) 39.x | 微信窗口自动化：发送文字、文件、导航联系人 |
| GUI 框架 | PyQt5 | 桌面窗口、三页签布局、大字体适老化 UI |
| 任务隔离 | `QProcess` 子进程 | 每次发送启动独立 Python 进程，COM 公寓完全隔离 |
| 系统交互 | psutil、winreg | 微信进程检测、安装路径五层降级自动发现 |
| 数据持久化 | JSON（tasks.json） | 定时任务队列本地存储，重启自动恢复 |

### 系统要求

- **操作系统**：Windows 10 / 11 / Server 2016+
- **Python**：3.7+（不支持 3.7.6 和 3.8.1）
- **微信版本**：3.9.x 系列（推荐 3.9.8.15 至最新版）
- **依赖**：`pip install wxauto PyQt5 psutil pywin32`

## 3. 架构详解

### 3.1 核心问题：COM 线程冲突

wxauto 底层依赖 Windows COM / UIAutomation，COM 对象绑定在创建它的线程公寓（Apartment）中。早期版本直接在 QThread 内调用 wxauto，导致 COM STA 与 PyQt 事件循环互斥，引发硬崩溃（ACCESS_VIOLATION）。

**最终方案：进程隔离**

```
GUI 主线程（零 COM 操作）
  │
  ├─ 用户点击 → QProcess 启动子进程
  │                │
  │                └→ auto_test.py → import wxauto → WeChat() → SendMsg/SendFiles
  │                                                                  │
  │                                                         独立 COM 公寓 ✅
  │
  └─ QTimer → check_scheduled_tasks() → QProcess → auto_test.py → ...
       │
       每个子进程拥有独立的 Python 解释器 + COM 公寓，互不干扰
```

### 3.2 SendWorker — 零线程任务调度

```python
class SendWorker(QObject):
    finished = pyqtSignal(str, bool)
    # parent=self 确保主窗口持有引用，防止 GC 回收
    # QProcess 异步运行，finished 信号在子进程退出后触发
```

**设计要点**：
- 继承 `QObject` 而非 `QThread`，所有代码运行在主线程
- `parent=self` 保证 Python + C++ 双引用，防止对象被 GC
- `self.deleteLater()` 在子进程完成后自动释放

### 3.3 auto_test.py — 独立子进程脚本

```python
# 零 PyQt5 依赖，纯 wxauto 脚本
from wxauto import WeChat
wx = WeChat()
wx.SendMsg(text, target)
wx.SendFiles(image_path, target)
```

**设计要点**：
- 不引入任何 PyQt5 / comtypes 依赖
- `sys.exit(0)` 表示成功，`sys.exit(1)` 表示失败
- `stdout` 输出发送结果，GUI 通过 `QProcess.readAllStandardOutput()` 读取

### 3.4 微信路径自动发现

`discover_wechat_path()` 五层降级策略：

| 优先级 | 策略 | 原理 |
|--------|------|------|
| 1 | 正在运行的进程 | `psutil` 定位 `wechat.exe` 完整路径 |
| 2 | 注册表扫描 | `HKCU\Software\Tencent\WeChat` 等三条注册表路径 |
| 3 | 开始菜单快捷方式 | 解析 `.lnk` 文件 |
| 4 | 常见安装目录 | C/D/E/F 盘常见路径硬搜索 |
| 5 | 系统 PATH | 在 PATH 中搜索 `WeChat.exe` |

### 3.5 定时任务数据模型

```python
class ScheduledTask:
    - id          # 自增 ID
    - recipient   # 联系人
    - message     # 文字内容
    - file_path   # 附件路径
    - send_time   # 预定发送时间
    - repeat      # 是否重复发送
    - interval    # 重复间隔（分钟）
    - status      # 状态机：等待中/发送中/已发送/发送失败
```

提供 `to_dict()` / `from_dict()` 支持 JSON 序列化。

## 5. 任务调度核心逻辑

```
QTimer(1000ms) → check_scheduled_tasks()
     ↓
遍历 scheduled_tasks，找到 status=="等待中" 且 send_time <= now
     ↓
标记"发送中" → SendWorker → QProcess → auto_test.py → wxauto → 微信
     ↓
on_scheduled_send_finished() 回调
     ├→ 重复任务：更新 send_time += interval，status 复位为"等待中"
     └→ 单次任务：status = "已发送"
     ↓
self._sch_worker = None  →  refresh_task_table()  →  save_tasks()
```

**互斥保护**：`self.sending_task_id` 确保同一时间仅一个任务在执行。

## 7. 常见问题排查

| 现象 | 原因 | 解决方案 |
|------|------|---------|
| 任务一直显示"发送中" | SendWorker 被 GC 回收，回调丢失 | 已修复：`parent=self` + `_sch_worker` 保持引用 |
| 闪退 | 旧版 QThread + COM 冲突 | 已修复：QProcess 子进程隔离 |
| 微信无反应 | 微信窗口被遮挡 | 保持微信窗口可见 |
| 定时任务过期消失 | 发送时间早于当前时间 | 设定未来的发送时间 |

## 8. 开发演进史

| 阶段 | 核心技术 | 问题 |
|------|---------|------|
| V1 自制 UIA | 手动 `uiautomation` + `SetFocus`/`SendKeys` | 焦点管理脆弱，子线程按键不可靠 |
| V2 多策略引擎 | SmartSendExecutor 三策略 + 剪贴板 | 策略切换崩溃，COM 线程冲突 |
| V3 wxauto + QThread | wxauto 库 + QThread + CoInitialize | COM STA 与 Qt 事件循环互斥 → 闪退 |
| V4 wxauto + QProcess | **当前版本**：QProcess 子进程隔离 | 稳定可靠 ✅ |
